# Sprint 1 Retrospective — The Velvet Room

**Sprint Duration:** Week 1    
**Date of Retrospective:** End of Week 1
**Facilitator:** Lee Wen Xin (Scrum Master)  
**Attendees:** Lee Wen Xin · Jaff Go · Ngoi Chang Zen · Lam Chuan Fong

---

## ✅ What Went Well

- **Task division was clear from the start.** Frontend (Jaff Go) and backend (Lee Wen Xin) responsibilities were separated early, which allowed both sides to develop in parallel without blocking each other.
- **Core features were delivered on schedule.** The reservation form, API endpoints, and database schema were all completed within the sprint, meeting the Sprint 1 goal.
- **Database schema design was solid.** The `room`, `package`, and `reservation` tables were structured cleanly with foreign key relationships, making Sprint 2 backend work straightforward to build on.
- **GitHub branching was used consistently.** Each member worked on separate feature branches, and no major merge conflicts occurred during Sprint 1.

---

## ⚠️ What Could Be Improved

- **Manual QA testing was not prioritised early enough.** Testing was left to the final day of the sprint, which left little time to identify and fix bugs before the sprint review.
- **Communication about API response formats was informal.** Frontend and backend had minor mismatches in expected JSON structure early on, causing small delays when connecting the form to the API.
- **Sprint planning did not include time estimates per task.** Without time estimates, it was difficult to gauge workload balance across team members during the sprint.

---

## 🔧 Actions Taken for Sprint 2

| Issue | Action |
|-------|--------|
| QA testing left too late | Dedicated QA time scheduled into Sprint 2 plan from day one; test cases written alongside development |
| API response mismatches | Frontend and backend agreed on JSON structure in writing before implementation began |
| No time estimates in tasks | Story Points and hour estimates added to all Sprint 2 tasks before sprint started |
