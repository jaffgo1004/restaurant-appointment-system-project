## Board Structure

The team used a 4-column Kanban board hosted on GitHub Projects to track all user stories across both sprints.

| Column | Description |
|--------|-------------|
| **Backlog** | User stories not yet started |
| **In Progress** | Actively being developed |
| **In Review** | Completed and under testing / review |
| **Done** | Fully completed and verified |

## Workflow Notes

- Each user story was created as a card on the board at the start of each sprint
- Cards were moved from **Backlog → In Progress → In Review → Done** as work progressed
- Sprint Reviews were conducted at the end of each sprint before closing all cards into Done
- The board was updated daily during active development weeks
- GitHub Issues were used alongside the board to track bugs and tasks at a more granular level
